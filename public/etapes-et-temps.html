4. Crée le repo

7. Création de l'App sur Coolify
"Utilise Coolify (Server UUID: xxx, Project UUID: yyy). Crée une nouvelle application liée au dépôt GitHub que nous venons de créer. Expose le port 5173 et utilise le paramètre nixpacks."
 L'agent traduit une intention métier ("crée une app liée à github") en une série d'appels API ou d'interactions MCP complexes vers l'infrastructure de Coolify pour provisionner numériquement le projet sur ton VPS. ~3 min
8. Déploiement sur VPS Hostinger
"Assigne le nom de domaine `builder.mondomaine.com` à cette application sur Coolify, puis lance un déploiement forcé. Surveille les statuts et dis-moi quand le domaine est accessible."
 C'est une commande multi-étapes. L'agent injecte d'abord le domaine (FQDN), déclenche l'action de déploiement (build du conteneur, certs SSL via Traefik), puis agit comme un moniteur en "pollant" (vérifiant) l'état de l'API Coolify jusqu'à ce que le statut passe à `finished` avant de te confirmer que tout est en ligne. ~4 min
